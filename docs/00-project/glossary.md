# Glossary

| Term | Meaning in CloudAILab |
|---|---|
| Agent under test | AI or deterministic automation evaluated by a scenario. |
| Attack path | A sequence of identity, trust, and permission relationships that reaches a protected action or resource. |
| Canonical graph | Provider-neutral representation of principals, resources, relationships, and policies. |
| Current state | The normalized state collected from all running provider backends. |
| Desired state | Resources and relationships declared by a scenario before intentional learner changes. |
| Evidence | Structured state or events that support a verification result. |
| Facade | A scoped API implementation shaped like part of an external provider. |
| Fidelity | The documented degree to which an operation matches provider behavior. |
| Ground truth | Hidden or protected scenario requirements used by deterministic verification. |
| Invariant | A condition that must remain true for a scenario to pass. |
| Principal | A human, workload, application, service account, service principal, or agent identity. |
| Provider adapter | Code that deploys, collects, and normalizes provider-specific state. |
| Scenario | A versioned enterprise topology, mission, vulnerabilities, events, and verification rules. |
| Tenant | An isolated organizational identity boundary. |
| Trust edge | A relationship that permits identity, authority, or data access to cross a boundary. |
