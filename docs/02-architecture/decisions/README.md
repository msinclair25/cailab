# Architecture decision records

ADRs record significant decisions that affect system structure, security, compatibility, or long-term maintenance.

## Index

- [ADR-0001: Canonical provider-neutral graph](0001-canonical-provider-neutral-graph.md)

## Lifecycle

`proposed` → `accepted` → `superseded` or `deprecated`

Accepted ADRs are not edited to disguise a changed decision. A new ADR supersedes the previous record and links back to it.
